# Инструкция по проверке логов бэкенда

## Шаг 1: Подключение к серверу по SSH

```bash
ssh root@188.244.115.152
```

Пароль: `Xtayzysvincere1_`

## Шаг 2: Переход в директорию проекта

```bash
cd /var/www/unicrew
```

## Шаг 3: Проверка логов бэкенда

### Вариант 1: Последние 100 строк логов
```bash
docker logs unicrew-backend-1 --tail 100
```

### Вариант 2: Мониторинг логов в реальном времени (рекомендуется)
```bash
docker logs unicrew-backend-1 --tail 100 -f
```
(Нажмите `Ctrl+C` чтобы выйти из режима мониторинга)

### Вариант 3: Поиск по ключевым словам
```bash
docker logs unicrew-backend-1 2>&1 | grep -E "НАЧАЛО|FILES|avatar|✓✓✓|✗✗✗"
```

### Вариант 4: Последние 200 строк с фильтрацией
```bash
docker logs unicrew-backend-1 --tail 200 | grep -A 30 "НАЧАЛО"
```

## Шаг 4: Проверка статуса контейнера

```bash
docker ps | grep backend
```

## Шаг 5: Перезапуск бэкенда (если нужно)

```bash
cd /var/www/unicrew
docker compose restart backend
```

## Шаг 6: Выход из SSH

```bash
exit
```

## Полная последовательность команд (скопируйте и выполните):


После выполнения команды `docker logs unicrew-backend-1 --tail 100 -f`:
- Оставьте терминал открытым
- Откройте браузер и попробуйте загрузить аватар
- В терминале появятся новые логи с сообщениями `=== НАЧАЛО ОБНОВЛЕНИЯ ПРОФИЛЯ ===`
- Нажмите `Ctrl+C` чтобы остановить мониторинг
