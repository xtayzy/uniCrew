1. ОПИСАНИЕ ПРОЕКТА
═══════════════════════════════════════════════════════════════════════════════

UniCrew.kz — это веб-платформа для поиска команды и участников для студенческих 
проектов. Платформа предназначена для студентов университетов, которые нуждаются 
в поиске единомышленников с определенными навыками для совместной работы над 
учебными проектами, дипломными работами, хакатонами, стартапами и другими 
командными заданиями.

Основная цель проекта — упростить процесс формирования студенческих команд 
через автоматизированную систему поиска и фильтрации участников по различным 
критериям: навыкам, личным качествам, факультету, курсу и другим параметрам.

Ключевые возможности платформы:
- Регистрация и авторизация пользователей с двухэтапной верификацией email
- Создание и редактирование профилей с указанием навыков, личных качеств, 
  факультета, курса и другой информации
- Поиск пользователей по множественным фильтрам (навыки, качества, школа, 
  факультет, курс, уровень образования)
- Создание команд с указанием требуемых навыков и качеств
- Система приглашений в команды с уведомлениями
- Система заявок на вступление в команды
- Управление участниками команд (принятие/отклонение заявок, изменение статусов)
- Система задач (Task Tracker) для отслеживания прогресса работы в команде
- Система уведомлений о всех важных событиях
- Публичные и приватные страницы команд
- Пагинация для оптимизации загрузки данных
- Адаптивный дизайн для всех типов устройств

3. АРХИТЕКТУРА ПРОЕКТА
═══════════════════════════════════════════════════════════════════════════════

Проект построен по архитектуре "клиент-сервер" с разделением на frontend и 
backend части:

BACKEND АРХИТЕКТУРА:
────────────────────
- Модель MVC (Model-View-Controller) через Django
- RESTful API архитектура
- Модели данных:
  * User (расширенная модель пользователя Django)
  * Team (команды проектов)
  * TeamMember (участники команд со статусами)
  * Skill, PersonalQuality (навыки и личные качества)
  * School, Faculty (школы и факультеты)
  * ProjectCategory (категории проектов)
  * Notification (система уведомлений)
  * Task (задачи в командах)
  
- ViewSets и API Views для обработки запросов
- Serializers для сериализации/десериализации данных
- Permissions для контроля доступа
- Pagination для оптимизации загрузки данных

FRONTEND АРХИТЕКТУРА:
─────────────────────
- Single Page Application (SPA) на React
- Компонентная архитектура
- Context API для управления состоянием (AuthContext, GlobalLoading)
- React Router для навигации
- Защищенные и публичные маршруты
- Модульные CSS стили
- Обработка ошибок через ErrorBoundary

9. СТРУКТУРА БАЗЫ ДАННЫХ
═══════════════════════════════════════════════════════════════════════════════

Основные модели:
────────────────
- User — пользователи системы (расширенная модель Django User)
- Team — команды проектов
- TeamMember — связь пользователей с командами (many-to-many через промежуточную 
  модель со статусами)
- Skill — навыки
- PersonalQuality — личные качества
- School — школы университета
- Faculty — факультеты (связь с школами)
- ProjectCategory — категории проектов
- Notification — уведомления
- Task — задачи в командах
- PendingUser — временные пользователи на этапе регистрации

Связи:
──────
- User имеет Many-to-Many связи с Skill и PersonalQuality
- User имеет ForeignKey к Faculty
- Team имеет ForeignKey к User (создатель) и ProjectCategory
- TeamMember связывает User и Team со статусами (PENDING, APPROVED, REJECTED)
- Notification связана с User и TeamMember
- Task связана с Team и User (исполнитель)


10. API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════

Аутентификация:
───────────────
- POST /api/register-step1/ — первый этап регистрации
- POST /api/register-step2/ — второй этап регистрации (верификация)
- POST /api/token/ — получение JWT токенов
- POST /api/token/refresh/ — обновление access токена

Пользователи:
─────────────
- GET /api/users/ — список пользователей (с пагинацией и фильтрацией)
- GET /api/profile/ — профиль текущего пользователя
- PATCH /api/profile/ — обновление профиля
- GET /api/users/{username}/ — детали пользователя

Команды:
────────
- GET /api/teams/ — список команд (с пагинацией и фильтрацией)
- POST /api/teams/ — создание команды
- GET /api/teams/{id}/ — детали команды
- PUT /api/teams/{id}/ — обновление команды
- DELETE /api/teams/{id}/ — удаление команды
- POST /api/teams/{id}/invite/ — приглашение пользователя
- POST /api/teams/{id}/approve/ — принятие заявки
- POST /api/teams/{id}/reject/ — отклонение заявки
- GET /api/teams/{id}/members/ — участники команды
- PUT /api/teams/{id}/members/{member_id}/ — изменение статуса участника
- GET /api/teams/{id}/requests/ — заявки на вступление
- GET /api/teams/{id}/tasks/ — задачи команды
- POST /api/teams/{id}/tasks/ — создание задачи
- PUT /api/teams/{id}/tasks/{task_id}/ — обновление задачи
- DELETE /api/teams/{id}/tasks/{task_id}/ — удаление задачи

Уведомления:
────────────
- GET /api/users/notifications/ — список уведомлений
- POST /api/users/mark_notification_read/ — отметка как прочитанное
- POST /api/users/mark_all_notifications_read/ — отметить все как прочитанные
- POST /api/users/accept_invitation/ — принятие приглашения
- POST /api/users/reject_invitation/ — отклонение приглашения
- POST /api/users/delete_notification/ — удаление уведомления

Справочники:
────────────
- GET /api/skills/ — список навыков (с поиском)
- GET /api/personal-qualities/ — список качеств (с поиском)
- GET /api/schools/ — список школ
- GET /api/faculties/ — список факультетов (с фильтрацией по школе)
- GET /api/project-categories/ — категории проектов
